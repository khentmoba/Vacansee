# Implementation Plan: Owner Property Management

**Branch**: `001-owner-property-management` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-owner-property-management/spec.md`

## Summary
Implement a secure, real-time property management suite for owners. This includes a pre-populated edit form for listing metadata and rooms, a deletion workflow with full asset cleanup (Storage + DB), and a re-verification trigger for address changes.

## Technical Context
- **Language/Version**: Dart 3.x / Flutter 3.x
- **Primary Dependencies**: `supabase_flutter`, `freezed`, `json_serializable`
- **Storage**: Supabase Postgres (Database), Supabase Storage (Images)
- **Testing**: `flutter test` (Unit/Widget)
- **Target Platform**: Flutter Web (Mobile-responsive)
- **Project Type**: Web application
- **Constraints**: $0 budget (Supabase Free Tier)

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Implementation Strategy |
|-----------|--------|--------------------------|
| I. Real-time Vacancy | ✅ | Use `supabase.from('properties').stream()` for instant sync. |
| II. Admin Verification | ✅ | Address changes set `verified = false`, triggering admin re-review. |
| III. Performant Simplicity | ✅ | Batch deletion of storage files and database rows in one transaction. |
| IV. Mobile-First | ✅ | Responsive Flutter layouts using `LayoutBuilder` and `OrientationBuilder`. |
| V. Type-Safe Sovereignty| ✅ | Generate models using Freezed; separate logic into `ListingService`. |
